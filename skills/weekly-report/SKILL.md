# weekly-report — 주간 업무 보고서 자동 생성

PIMS 시간 데이터와 git 커밋 이력을 결합하여 **주간 업무 보고서를 자동 생성**하고, 사용자 확인 후 PIMS에 제출합니다.

## Usage

```bash
/weekly-report              # 이번 주 보고서 생성
/weekly-report --dry-run    # 보고서 초안만 출력 (PIMS 제출 없음)
/weekly-report history      # 과거 보고서 이력 조회
/weekly-report 2026-03-31   # 특정 주 보고서 생성 (해당 날짜가 포함된 주)
```

**파라미터:**
- `history` — 과거 주간 보고서 이력 조회
- `--dry-run` — PIMS 제출 없이 보고서 초안만 출력
- `[날짜]` — (선택) 특정 주 기준 날짜. 생략 시 이번 주

**출력:**
- 주간 업무 보고서 (마크다운)
- PIMS 제출 결과
- 로컬 파일 저장 경로

---

## Instructions

당신은 주간 업무 보고서를 자동 작성하는 어시스턴트입니다.
PIMS 시간 데이터와 git 이력을 결합하여 정확하고 포괄적인 보고서를 생성합니다.

### 1단계: 주간 범위 결정

```
기준 날짜로부터 해당 주의 월요일~일요일 범위를 계산합니다.

예: 2026-04-07 (월) → 2026-04-07 ~ 2026-04-13
예: 2026-04-09 (수) → 2026-04-07 ~ 2026-04-13
```

### 2단계: PIMS 시간 데이터 수집

```
1. 주간 시간 요약:
   인자 없음 (이번 주): mcp__pims__get_time_summary(period="week")
   특정 날짜 지정 시:  mcp__pims__get_time_summary(from={월요일}, to={일요일})

2. 일별 상세 내역:
   mcp__pims__get_daily_time(date={각 날짜})
   — 월~금 각 날짜별로 호출

3. 내 담당 이슈 상태:
   mcp__pims__get_my_issues()
   — 이번 주에 진행/완료한 이슈 필터링
```

### 3단계: git 이력 수집

```bash
# 이번 주 커밋 목록
git log --since="{월요일} 00:00:00" --until="{일요일} 23:59:59" --pretty=format:"%h|%ai|%s" --no-merges

# 변경 파일 통계
git log --since="{월요일} 00:00:00" --until="{일요일} 23:59:59" --pretty=format:"" --shortstat --no-merges
```

```
수집 항목:
  - 총 커밋 수
  - 커밋 메시지에서 티켓 번호 추출 (#숫자)
  - 변경 파일 수 / 삽입 / 삭제 라인 수
```

### 4단계: 보고서 초안 생성

아래 형식으로 보고서를 작성합니다:

```markdown
# 주간 업무 보고 ({월요일} ~ {금요일})

## 시간 요약
| 활동 | 시간 | 비율 |
|------|------|------|
| R&D | {N}h | {N}% |
| 회의 | {N}h | {N}% |
| 코드리뷰 | {N}h | {N}% |
| 기타 | {N}h | {N}% |
| **합계** | **{N}h** | **100%** |

## 일별 시간
| 요일 | 시간 | 주요 활동 |
|------|------|----------|
| 월 | {N}h | {이슈 제목} |
| 화 | {N}h | {이슈 제목} |
| 수 | {N}h | {이슈 제목} |
| 목 | {N}h | {이슈 제목} |
| 금 | {N}h | {이슈 제목} |

## 주요 성과
{이번 주 완료/진행한 이슈 목록. PIMS 상태와 git 커밋을 교차 확인하여 실제 작업 내용 기술}
- #118746 KCM 에이전트 목록 BE 개발 완료 (진행률 100%)
- #118815 에이전트 상세 FE 화면 생성 착수 (진행률 30%)

## git 활동
- 커밋: {N}건
- 변경: {파일수} files, +{삽입}/-{삭제} lines

## 다음 주 계획
{진행중 이슈 + 신규 이슈 기반으로 다음 주 예상 작업 나열}
- #118815 에이전트 상세 FE 완료 예정
- #118900 알람 API 개발 착수 예정

## 이슈/리스크
{기한 임박 이슈, 블로커, 미해결 문제 등}
- (없으면 "특이사항 없음")
```

### 5단계: 사용자 검토

```
생성된 보고서 초안을 사용자에게 표시합니다.

"주간 보고서 초안이 생성되었습니다.

 수정할 내용이 있으면 알려주세요.
 - '확인' → PIMS에 제출합니다
 - '수정: {내용}' → 수정 후 다시 확인합니다
 - '취소' → 제출하지 않습니다"

--dry-run 모드인 경우:
  초안을 출력하고 종료합니다. PIMS 제출 단계로 넘어가지 않습니다.
```

### 6단계: PIMS 제출 (Retry-with-Confirmation)

```
사용자가 확인하면 PIMS에 제출합니다:

1. dry-run 출력: "PIMS에 주간 보고서를 제출합니다."
2. 사용자 최종 확인
3. mcp__pims__weekly_report(...)
4. 성공 시: "PIMS 주간 보고서 제출 완료"
5. 실패 시:
   - 네트워크/타임아웃: "재시도할까요?" (최대 2회)
   - 권한 오류: "권한이 없습니다. 수동으로 처리하세요."
   - 기타: 원본 에러 출력 + 수동 처리 가이드
```

### 7단계: 로컬 파일 저장

```
보고서를 로컬에도 저장합니다:

경로: {output_dir}/weekly-report-{YYYYMMDD}.md
  - {output_dir}는 `.claude/skill-config.yaml`의 `weekly_report.output_dir` 값
  - 설정이 없으면 기본값: docs/weekly-reports/
  - 날짜는 해당 주 월요일 기준

디렉토리가 없으면 생성합니다.

※ .claude/skill-config.yaml 예시:
  weekly_report:
    output_dir: "docs/weekly-reports/"
```

### history 모드: 과거 보고서 조회

```
/weekly-report history 실행 시:

mcp__pims__get_weekly_report_history()

결과를 테이블로 표시:
| # | 기간 | 제출일 | 총 시간 |
|---|------|--------|---------|
| 1 | 03-31 ~ 04-04 | 2026-04-04 | 40h |
| 2 | 03-24 ~ 03-28 | 2026-03-28 | 38h |
```

## 규칙

- PIMS 제출 전 반드시 사용자 확인 필요
- `--dry-run` 모드에서는 PIMS API 쓰기 호출 절대 없음
- PIMS 시간 데이터가 없으면 git 이력만으로 보고서 생성 (경고 표시)
- git 커밋이 없으면 PIMS 데이터만으로 보고서 생성
- 모든 텍스트는 한국어로 출력
- 보고서는 로컬 파일로도 반드시 저장
