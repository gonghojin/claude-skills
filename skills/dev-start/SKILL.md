# dev-start — 개발 시작 대시보드

하루 개발 시작 시 **리마인더, 내 담당 이슈, 어제 작업 요약**을 한눈에 보여주는 대시보드를 출력합니다.

## Usage

```bash
/dev-start           # 오늘 기준 대시보드
/dev-start 2026-04-07  # 특정 날짜 기준 대시보드
```

**파라미터:**
- `[날짜]` — (선택) 기준 날짜. 생략 시 오늘

**출력:**
- 리마인더 목록
- 내 진행중/신규 이슈 목록
- 어제 작업 요약 (시간 기록 + git 커밋)
- 추천 다음 작업

---

## Instructions

당신은 개발자의 하루를 돕는 어시스턴트입니다.
PIMS 데이터와 git 이력을 결합하여 개발 시작에 필요한 정보를 대시보드로 제공합니다.

### 1단계: 리마인더 확인

```
mcp__pims__check_reminders()

확인 항목:
  - 기한 임박 이슈 (오늘/내일 마감)
  - 오버듀 이슈 (마감일 초과)
  - 기타 리마인더

리마인더가 없으면 "리마인더 없음"으로 표시 (오류 아님)
```

### 2단계: 내 담당 이슈 조회

```
mcp__pims__get_my_issues()

필터링:
  - 상태: 신규(1), 진행(4) — 완료/종료/보류 제외
  - 정렬: 우선순위 높음 > 진행률 낮음 > 마감일 임박

표시 정보:
  - 티켓 번호 + 제목
  - 상태 (신규/진행중)
  - 진행률 (%)
  - 우선순위
  - 마감일 (D-N 형태)
```

### 3단계: 어제 시간 기록 확인

```
어제 날짜 계산 (인자로 날짜가 주어지면 그 전날):

mcp__pims__get_daily_time(date={어제_날짜})

확인 항목:
  - 어제 기록된 시간 합계
  - 이슈별 기록 시간
  - 활동 유형별 분류 (개발, 회의, 리뷰 등)
```

### 4단계: 어제 git 커밋 수집

```bash
git log --since="{어제} 00:00:00" --until="{오늘} 00:00:00" --pretty=format:"%h|%ai|%s" --no-merges
```

```
수집 항목:
  - 커밋 수
  - 커밋 메시지 목록
  - 커밋 메시지에서 티켓 번호 추출 (#숫자 패턴)

미기록 커밋 감지:
  - git 커밋은 있지만 PIMS 시간 기록이 없는 경우
  - "어제 시간 미기록 커밋이 있습니다" 알림 + /log-work 안내
```

### 5단계: 대시보드 출력

아래 형식으로 대시보드를 출력합니다:

```markdown
# 오늘의 개발 현황 ({날짜})

## 리마인더
{리마인더가 있으면 목록, 없으면 "리마인더 없음"}
- ⚠️ #118900 [BE] 알람 API 개발 — 마감 D-1
- 🔴 #118850 코드 리뷰 — 마감일 초과 (D+2)

## 내 이슈 ({N}건)
| # | 티켓 | 상태 | 진행률 | 우선순위 | 마감 |
|---|------|------|--------|----------|------|
| 1 | #118746 KCM 에이전트 BE | 진행중 | 70% | 높음 | D-5 |
| 2 | #118815 에이전트 상세 FE | 신규 | 0% | 중간 | D-10 |

## 어제 작업 요약
- 커밋: {N}건
- PIMS 기록: {N}h ({활동유형별 요약})
{미기록 커밋이 있으면:}
- ⚠️ 시간 미기록 커밋 {N}건 — `/log-work`로 기록하세요

## 추천 다음 작업
{우선순위 + 진행률 + 마감일 기반으로 1~3건 추천}
1. #118900 [BE] 알람 API 개발 — 마감 임박(D-1), 진행률 30%
2. #118815 에이전트 상세 FE — 신규, 진행률 0%
```

### 추천 다음 작업 선정 기준

```
1순위: 마감일 임박 + 진행률 낮음 (위험 항목)
2순위: 진행중 + 진행률 낮음 (작업 재개 필요)
3순위: 신규 + 우선순위 높음 (새 작업 시작)

최대 3건까지 추천
```

---

## 규칙

- **조회 전용** — PIMS에 데이터를 쓰거나 변경하지 않음
- PIMS API 호출 실패 시 해당 섹션만 "데이터 조회 실패" 표시, 나머지 섹션은 정상 출력
- git log가 없으면 "어제 커밋 없음" 표시
- 모든 텍스트는 한국어로 출력
